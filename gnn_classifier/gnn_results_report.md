# GNN 黑名單分類實驗報告

## 實驗目的

比較兩種圖邊建構策略對 GNN 黑名單分類效果的影響：

- **internal-only**：只用平台內部轉帳（`sub_kind=1`，4,473 筆，佔 3%）
- **all**：內部轉帳 + 鏈上交易共享錢包橋接（`sub_kind=0`，235,485 筆，佔 97%）

---

## 鏈上交易邊的建構邏輯（共享錢包橋接）

原始鏈上交易的對手方是外部錢包 hash，不是平台用戶，無法直接建立 user-to-user 邊。

採用**共享錢包橋接**策略：
> 若 user A 出金到 wallet W（`kind=1, sub_kind=0`），且 user B 從 wallet W 入金（`kind=0, sub_kind=0`），則在 A 和 B 之間建立一條邊。

這樣可以捕捉「A 出金到某錢包，B 從同一錢包入金」的洗錢中轉模式。

---

## 圖結構統計

| 模式 | 內部轉帳邊 | 鏈上橋接邊 | 總邊數（去重） |
|------|-----------|-----------|--------------|
| internal | 1,446 | 0 | 1,446 |
| all | 1,446 | 4,739 | **6,138** |

- 涉及出金 user：31,930 人
- 涉及入金 user：18,302 人
- 節點總數：51,017（有標籤用戶）
- 正常 / 黑名單：49,377 / 1,640（比例約 30:1）

## 關鍵發現

### 1. 加入鏈上橋接邊後效果明顯提升

| 指標 | internal | all | 提升幅度 |
|------|---------|-----|---------|
| AUC-ROC | 0.505 | 0.541 | +7.1% |
| AUC-PR | 0.038 | 0.051 | +34.2% |
| F1（黑名單） | 0.037 | 0.103 | +178% |
| 黑名單召回率 | 3% | 12% | +4倍 |

共享錢包橋接策略有效，說明黑名單用戶確實存在透過共同外部錢包進行資金中轉的行為。

### 2. internal-only 幾乎無效

internal mode 的 AUC-ROC ≈ 0.505，接近隨機猜測（0.5）。原因：
- 只有 1,446 條邊，51,017 個節點，圖極度稀疏
- 大多數用戶是孤立節點，GNN 無法傳遞鄰居資訊
- 節點特徵（度數）幾乎全為零，模型退化為只看標籤分布

### 3. 整體效果仍有提升空間

GraphSAGE all 的 AUC-ROC 0.541 仍然偏低，主要原因：
- 黑名單比例極低（3.2%），嚴重類別不平衡
- 6,138 條邊對 51,017 個節點來說仍然稀疏（平均度數 < 0.25）
- 節點特徵只有度數相關 6 個特徵，缺乏交易金額、時間等豐富資訊

## 與其他方法的比較

| 方法 | AUC-ROC | 備註 |
|------|---------|------|
| ΔT1 Logistic Regression | 0.681 | 時間差特徵，目前最佳 |
| GNN GraphSAGE (all) | 0.541 | 圖結構 + 鏈上橋接 |
| GNN GCN (all) | 0.540 | 圖結構 + 鏈上橋接 |
| GNN (internal-only) | 0.505 | 接近隨機 |

---

## 改進方向

1. **豐富節點特徵**：加入 ΔT1/ΔT2、交易金額統計、幣種多樣性、IP 共享等特徵
2. **異質圖（HeteroGNN）**：把 wallet hash 也當節點，建立 user → wallet → user 的異質圖，可以利用全部 235,485 筆鏈上交易
3. **特徵融合**：將 GNN embedding 與 ΔT1、NMF 等特徵結合，用 LightGBM 做最終分類
4. **邊特徵**：加入交易金額、時間差等邊屬性，讓 GNN 學習更豐富的關係
