# U 型迴轉特徵分類模型訓練報告

## 📊 專案概述

本報告展示使用「脈衝次數」和「U 型迴轉比率」兩個特徵來識別人頭戶（黑名單用戶）的分類模型訓練結果。

### 特徵說明

1. **脈衝次數 (Pulse Count)**
   - 定義：帳戶餘額從接近零突然拉高，然後又快速歸零的次數
   - 意義：人頭戶的典型特徵是「過水」，資金不停留，表現為脈衝式的餘額變化
   - 計算方式：檢測餘額時間線中「低-高-低」的模式

2. **U 型迴轉比率 (U-Turn Ratio)**
   - 定義：大額交易後 24 小時內餘額歸零的比率
   - 意義：衡量資金停留時間，人頭戶通常在大額入金後快速提領
   - 計算方式：(24小時內歸零的大額交易數) / (總交易數)

## 🔍 模型性能分析

### 優勢

1. **高 Recall (52.97%)**
   - 能夠識別超過一半的黑名單用戶
   - 對於風控場景，高 Recall 意味著能抓到更多可疑帳戶

2. **穩定的交叉驗證表現**
   - CV F1 Mean: 0.1639 (± 0.0127)
   - 標準差小，表示模型穩定性好

3. **良好的 ROC-AUC (68.70%)**
   - 顯著優於隨機猜測 (50%)
   - 模型具有一定的區分能力

### 挑戰

1. **低 Precision (10.46%)**
   - 預測為黑名單的用戶中，只有約 10% 真正是黑名單
   - 會產生較多誤報（假陽性）

2. **類別不平衡問題**
   - 黑名單用戶只佔 5%
   - 導致模型傾向於預測為正常用戶

3. **F1-Score 偏低 (17.47%)**
   - Precision 和 Recall 之間存在權衡
   - 需要根據業務需求調整閾值

## 💡 業務應用建議

### 1. 風險分級策略

根據模型預測概率，建議採用三級風險分級：

| 風險等級 | 預測概率範圍 | 建議動作 |
|---------|-------------|---------|
| 🔴 **高風險** | > 0.7 | 立即人工審核，暫停大額交易 |
| 🟡 **中風險** | 0.3 - 0.7 | 加強監控，限制單日交易額度 |
| 🟢 **低風險** | < 0.3 | 正常交易，定期抽查 |

### 2. 閾值調整

當前模型使用預設閾值 0.5，可根據業務需求調整：

- **提高 Precision（減少誤報）**: 提高閾值至 0.6-0.7
  - 適用場景：避免誤傷正常用戶
  
- **提高 Recall（抓更多可疑帳戶）**: 降低閾值至 0.3-0.4
  - 適用場景：嚴格風控，寧可錯殺不可放過

### 3. 人工審核流程

建議將模型作為初篩工具：
1. 模型標記高風險用戶
2. 人工審核確認
3. 收集審核結果作為新訓練數據
4. 定期重新訓練模型

## 📝 結論

本次實驗成功使用「脈衝次數」和「U 型迴轉比率」兩個特徵訓練了人頭戶識別模型。**LightGBM** 模型表現最佳，能夠識別超過一半的黑名單用戶（Recall = 52.97%），ROC-AUC 達到 68.70%。

雖然 Precision 較低（10.46%），但這在高度不平衡的數據集中是常見的。建議將模型作為初篩工具，結合人工審核來提高整體風控效果。

透過持續的特徵工程、模型優化和數據收集，預期可以進一步提升模型性能。
