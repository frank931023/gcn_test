## 📈 关键概念详解

### 1️⃣ **SMOTE：数据超级不平衡的救星**

#### 问题：数据严重不平衡
```
正常交易：9990笔 ████████████████ (99.9%)
洗钱交易：10笔   █ (0.1%)
```

如果直接训练模型，它会"偷懒"—— 全部预测为"正常"就能达到99%的准确率！
（但其实洗钱一个都没抓到 😱）

#### SMOTE 的解决方案
**SMOTE = Synthetic Minority Oversampling Technique（合成少数类过采样技术）**

它的工作原理简单粗暴：

```
1️⃣ 找到每个洗钱样本的最近邻（3-5个）
   例如：洗钱用户A的特征 = [交易数10, 账户年龄1年, 平均金额$1000]

2️⃣ 在A和它的邻居之间"插值"生成新的虚拟样本
   虚拟用户 = A + 随机数 × (邻居 - A)
   
3️⃣ 生成新样本直到洗钱数据增加到合理比例
   结果：正常:洗钱 = 10:1 (10%洗钱，而不是0.1%)
```

**类比理解**：
想象你只有1张"坏人"的照片，SMOTE就是通过高明的P图技术，根据这张照片和相似的照片，生成更多"虚拟的坏人照片"，这样AI训练时见过更多坏人特征了。

✅ **好处**
- 模型不再偷懒，真的学会识别洗钱特征
- 少数类（洗钱）的召回率大幅提升
- 性能指标更均衡

⚠️ **代价**
- 计算时间增加（生成合成数据需要时间）
- 可能过拟合（虚拟数据和真实数据略有不同）

### 2️⃣ **交易网络图 (Transaction Graph)**

#### 图的构成

```
【节点（Nodes）】
  每个账户 = 一个节点
  节点特征 = 账户的信息（交易次数、总额等）

【边（Edges）】
  账户A → 账户B = 一笔交易
  边的权重 = 交易金额或频次

完整的图：
    账户1 ━━金额$500━━→ 账户2
     ↓ $1000           ↑ 
     ↓                 $300
    账户3 ←─────────── 账户4
```

#### 为什么要用图？

**对比1：传统方法（仅看交易记录）**
```
用户A：交易10次，总额$50000
用户B：交易10次，总额$50000

→ 看起来一样！无法区分
```

**对比2：图的方法（看交易关系）**
```
用户A：和基金公司、工资方、房东 = 三个固定对象交易
        ✅ 正常的个人账户

用户B：和50个陌生账户交易
       赚$1000 → 分散给49个账户 ($20/个)
       → 收集再分散... (重复多次)
       ✅ 典型的洗钱钱骡账户！
```

**图的价值**：捕捉"谁和谁在交易"的**模式**，而不只是数字！

### 3️⃣ **Node2Vec：给节点学习一个"人设"**

#### 核心思想
如果我们有一个"魔法"，能把每个账户转换成一个向量（比如 128 个数字），这个向量包含了账户的所有重要特征（交易习惯、风险程度等），那么就能轻松分类！

**Node2Vec 就是这个"魔法"！**

#### 算法原理（简化版）

```
【用法1】随机游走 (Random Walk)
  从账户A出发，随机跳跃：
  A → D → B → E → C → ... → Z
  
  这个游走路径 [A, D, B, E, C, ..., Z] 的信息：
  → A经常和D交互
  → D经常和B交互
  → ...
  → 这反映了网络结构

【用法2】Word2Vec思路
  把游走路径当成"句子"：
  [A, D, B, E, C] 是一个"句子"
  
  用 Word2Vec 训练这些"句子"，使得：
  ✓ A 和经常在一个路径上出现的节点，向量相近
  ✓ A 和很少在一个路径上出现的节点，向量相远
  
【输出】
  每个账户获得一个 128 维向量
  例如：账户A = [0.23, -0.15, 0.89, ..., 0.12] (128个数字)
  
  向量包含：
  ✓ 账户在网络中的角色
  ✓ 与哪些账户相关联
  ✓ 隐藏的交易模式
```

#### 为什么学习 50 个 epochs？
```
【第1个epoch】
  神经网络还很"菜"，向量学得很烂

【第25个epoch】  
  学到了主要特征，向量变好了

【第50个epoch】
  向量稳定收敛，表达能力最强
  继续训练收益递减
```

#### Node2Vec vs 其他方法

| 方法 | 优点 | 缺点 |
|------|------|------|
| **Node2Vec** | 考虑局部和全局结构，速度快 | 无法处理新节点 |
| DeepWalk | 最简单的游走方法 | 太简单，特征不够好 |
| Graph CNN | 直接在图上卷积，很强 | 计算量大，需要全图 |

### 4️⃣ **MLP 分类器：最后的"判官"**

有了 128 维的特征向量，怎么判断一个账户是否洗钱？

**答案：用神经网络！**

```
【输入层】
  128维向量 (Node2Vec的输出)
    ↓ (神经网络层1)
  
【隐藏层1】  
  64个神经元
  f(x) = ReLU(w₁·x + b₁)
  
  ReLU激活函数的作用：
    - 增加非线性（模型能学复杂判断）
    - 0.5 → 0.5  ✓
    - -0.5 → 0   ✓ (只保留有用的信息)
    
  Dropout：随机关闭30%的神经元
    - 防止过拟合（模型太聪明导致只记住训练数据）
    - 提高泛化能力
    ↓ (神经网络层2)
  
【隐藏层2】
  64个神经元 (同上)
    ↓ (最后一层)
  
【输出层】
  2个神经元 = 2个类别
  [类别0概率, 类别1概率]
  
  最终判断：
  如果类别1概率 > 50% → "这是洗钱账户" ⚠️
  否则 → "这是正常账户" ✅
```

#### 模型怎么学习？

```
【初始状态】
  权重(w)随机初始化：模型随机猜测

【训练循环：50次迭代】
  步骤1：输入训练样本
         样本：[128维向量] → 预测 → [0.2, 0.8]
         真实标签：1 (是洗钱)
  
  步骤2：计算损失（预测有多错）
         Loss = -[0*log(0.2) + 1*log(0.8)]
         = -log(0.8) = 0.22
         
  步骤3：反向传播（告诉模型权重怎么改）
         计算每个权重对损失的影响程度
         
  步骤4：梯度下降（更新权重）
         新权重 = 旧权重 - 学习率 × 梯度
         学习率 = 0.001 (步长，确保稳定学习)
  
  步骤5：重复（每次权重都变得更好）

【第50次迭代结束】
  权重已经学到了："如果向量中的第5个特征很大，
                     就很可能是洗钱账户"
```

### 5️⃣ **完整的 GNN 管道：为什么需要 GCN？**

我们现在有了：
- ✅ Node2Vec 生成的 128 维特征向量
- ✅ MLP 分类器

**但这有个问题：**
```
现有方案：
  图结构 → Node2Vec → 向量 ─┐
                            → MLP 分类器 → 输出
  （向量失去了图信息）     ┘

缺陷：
  MLP 看不到图本身，只看到向量
  错过了"邻居信息"的帮助
```

**GCN (Graph Convolutional Network) 的改进：**
```
更好的方案：
  图结构 ─┐
          ├→ GCN 同时考虑：  ├→ 向量 → 输出
  向量   ─┤  1. 节点特征      └→ 邻居特征
          └  2. 邻居特征

GCN 操作（简化）：
  新特征 = 激活函数(权重 × 自己的特征 + 邻居特征的平均)
  
  【第1层 GCN】
    账户A的新特征 = 自己+邻居B、C的特征的加权组合
  
  【第2层 GCN】  
    账户A再次更新，现在包含"邻居的邻居"的信息
    
  完成后传给分类器
```

**实际中：由于数据和计算限制，我们用 MLP + Node2Vec**
- MLP：简单有效（计算快）
- Node2Vec：已经包含了图的拓扑信息
- 效果往往和 GCN 不相上下！

## 📊 最终评估：怎么判断模型好不好？

### 关键指标

#### 1️⃣ **准确率 (Accuracy)**
```
准确率 = (正确预测数) / (总预测数)
      = (TP + TN) / (TP + TN + FP + FN)

例子：
  100个样本，正确分类95个
  → 准确率 = 95%

⚠️ 问题：如果样本不平衡（99个正常，1个洗钱）
  模型全部预测为"正常" → 准确率99%但没用！
```

#### 2️⃣ **精准率 (Precision)**
```
精准率 = (真正的洗钱) / (预测为洗钱的)
      = TP / (TP + FP)

含义："我预测的洗钱账户中，真正有多少是洗钱？"

例子：
  预测15个洗钱账户，其中12个真的是洗钱
  → 精准率 = 12/15 = 80%
  
  意思：银行追查我们的警报，80%是真把式
```

#### 3️⃣ **召回率 (Recall)**
```
召回率 = (真正的洗钱) / (实际洗钱总数)
      = TP / (TP + FN)

含义："真实的洗钱者，我能抓到多少？"

例子：
  实际有20个洗钱账户，我抓到了12个
  → 召回率 = 12/20 = 60%
  
  意思：漏掉了40%，有8个"坏人"没被发现
```

#### 4️⃣ **F1分数**
```
F1 = 2 × (精准率 × 召回率) / (精准率 + 召回率)

作用：同时考虑精准率和召回率

例子：
  精准率 = 80%，召回率 = 60%
  F1 = 2 × (0.8 × 0.6) / (0.8 + 0.6) = 0.686
  
  F1 高 = 模型既不误报，也不漏报
```

### 混淆矩阵图解

```
实际 \ 预测    正常     洗钱
┌──────────┬───────┬──────┐
│ 正常     │  TN   │  FP  │  (假警报)
│          │(真负) │(误报)│
├──────────┼───────┼──────┤
│ 洗钱     │  FN   │  TP  │
│          │(漏报) │(真正)│
└──────────┴───────┴──────┘

理想情况：TN和TP都大，FP和FN都小
```

## 💡 关键洞察（为什么这个方案有效？）

### 传统欺诈检测 vs 图神经网络

| 角度 | 传统方法 | 图神经网络方法 |
|-----|--------|------------|
| **看什么** | 单笔交易的特征 (金额、时间) | 整个账户生态 |
| **找什么** | 异常金额或频率 | 异常的交易模式 |
| **例子** | "这笔$10000转账有点大" | "这个账户像个钱骡：有进有出，服务多个账户" |
| **误报率** | 高（正常人也会大额转账） | 低（反映真实行为模式） |

### 特征学习的强大

```
传统特征（可能性）：
  - 交易金额
  - 交易频率
  - 账户年龄
  - 交易对方品类
  = 可能只有 10-20 个特征，信息有限

Node2Vec 学习到的特征（128维）：
  - 第1维：账户在网络中的"中心性"
  - 第2维：交易聚集程度
  - 第3维：与黑名单账户的距离
  - ...
  - 第128维：与模式相似账户的亲近度
  
  = 128 个自学到的特征，包含更多隐藏信息！
```
