4. 交易介面狀態轉移 (Markov Chain Transition)理論背景： 惡意造市商或機器人刷單，其操作路徑是寫死的腳本。人類在使用 APP 或 WEB 時，會有隨機的點擊延遲與頁面跳轉，而機器人往往只固定敲擊 API。提取欄位： user_info 的註冊來源 user_source 與 usdt_twd_trading 的訂單來源 source (0 WEB, 1 APP, 2 API) 。特徵計算： 計算用戶使用介面的切換機率。例如：註冊用 APP，但 100% 的交易都來自 API。視覺化呈現：狀態轉移圖 (State Transition Graph)。將 WEB、APP、API 設為節點，連線粗細代表切換頻率。視覺特徵： 如果某個帳戶的圖表呈現「只有 API 節點在自我無限迴圈」，且完全沒有其他節點的互動，這幾乎可以斷定是非人類行為。這類特徵對於訓練偵測虛擬資產異常活動的模型來說，是非常高權重的決策節點。
